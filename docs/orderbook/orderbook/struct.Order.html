Expand description

A single order tracked by the order book.

Tracks identity, side, price, and quantity lifecycle: initial → remaining/filled, with a convenience flag filled.

Fields§

§order_type: OrderType

Limit/market/GTC classification for matching behavior.

§order_id: u32

Unique identifier assigned by the client/system.

§side: Side

Buy or Sell.

§price: i32

Limit price. For market orders created via Order::new_market, this is initialized to a sentinel and may later be set by Order::to_good_till_cancel.

§initial_quantity: u32

Quantity at creation time.

§remaining_quantity: u32

Shares/contracts not yet executed.

§filled_quantity: u32

Cumulative executed size.

§filled: bool

Convenience flag set when remaining_quantity == 0.

pub fn fill(&mut self, quantity: u32) -> Result<(), String>

Applies a partial or full fill to the order.

Decrements remaining_quantity and increments filled_quantity. Sets filled = true when remaining_quantity reaches zero.
